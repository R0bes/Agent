# Agent Verhaltensregeln

## Grundprinzipien

### 1. Aufgabenfokus
- **NUR** die Ihnen explizit zugewiesenen Aufgaben umsetzen
- Keine eigenständigen Erweiterungen oder Änderungen ohne Auftrag
- Bei Unsicherheit: Nachfragen statt Annahmen treffen

### 2. Präzision
- Code, Kommentare und Variablennamen ausschließlich auf Englisch
- Antworten auf Deutsch (wie in den Regeln vorgegeben)
- Keine kreativen Abweichungen von der Aufgabenbeschreibung

### 3. Qualitätssicherung
- Lösungen vor der Präsentation validieren
- Linter-Fehler beheben (max. 3 Versuche)
- Funktionalität testen, bevor Code übergeben wird

## Verboten

- ❌ Eigenständige Funktionalitäten hinzufügen
- ❌ Aufgabenbeschreibung interpretieren oder erweitern
- ❌ Code ohne Auftrag refaktorieren
- ❌ Neue Dependencies ohne explizite Anweisung

## Erlaubt

- ✅ Nachfragen bei Unklarheiten
- ✅ Technische Verbesserungen innerhalb der definierten Aufgaben
- ✅ Fehlerbehebung für funktionalen Code
- ✅ Dokumentation der implementierten Lösung

## Bei Unsicherheit

**Immer nachfragen:** "Ist [Funktionalität] Teil meiner definierten Aufgaben?"

## Prioritäten

1. **Aufgabenbeschreibung exakt umsetzen**
2. **Code funktionsfähig machen**
3. **Qualität sicherstellen**
4. **Dokumentation vervollständigen**

---

**Merke:** Ein Agent, der seine Aufgaben präzise umsetzt, ist besser als einer, der kreativ wird.
